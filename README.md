# Общая рекомендация
1. Завести отдельную папку, в которой будут находиться хранилища.
    - Например, в Документах создать папку Obsidian или типо того.
2. Если стоит английский язык:
    - Настройки (в открытом хранилище, снизу ближе к левой стороне) > General (уже открыт при открытии настроек) > Language - Русский > Relaunch
3. Настройки > Редактор > Отображение > Ограничить максимальную длину строки - При отключении текст становится шире.
4. При чтении файлов использовать Режим чтения. Его можно включить сверху справа в основной вкладке (где открываются файлы), кнопка с троеточием > у параметра "Просмотр" должна стоять галочка.
    - Не критично, если вы испортили что-то в файле. Нужно всего лишь ввести `git restore .` в открытой папке хранилища (после прочтения остального текста будет понятно, как это делать. Полностью аналогично Второму пути синхронизации).
5. Чтобы больше узнать об Obsidian или если что-то непонятно вы можете обратиться лично ко мне.

# Скачивание хранилища
1. Открыть cmd
    - Win+R, кто не шарит
2. Открыть в cmd папку, в которой будет хранилище.
    - Папка Obsidian в Документах, если следовать моей рекомендации
        - Это можно сделать легко так: Написать cd в консоли (не забываем пробел поставить после этого) > Перетащить нужную папку в консоль > Enter > Profit
    - Более подробно:
        - По умолчанию консоль открывается в папке C:\Users\ИмяПользователя.
        - `cd` - как понятно, нужен для навигации.
        - `cd названиеПапки` - перейти в папку с этим названием в текущей папке. Не чувствителен к РеГиСтРу.
            - Например, сразу после открытия консоли `cd documents`, откроется папка Документы (C:\Users\имяПользователя\Documents).
        - `cd ..` - подняться на уровень выше.
            - Например, после написания `cd ..` в папке Документы обратно откроется папка пользователя (C:\Users\имяПользователя).
3. Ввести в консоль `git clone ссылка` (где ссылка - ссылка на репозиторий. Например, https://github.com/Wudrus/GreatFortressModdingMD, репозиторий с нашим моддингом TF2)
4. Хранилище со всеми файлами на момент скачивания установлено. Есть два путя для дальнейшей синхронизации файлов.
## Путь первый - Obsidian Git - Простой и автоматический
1. Открываем нужное хранилище.
2. Настройки > Сторонние плагины > Включить плагины сообщества > Обзор > Ищем Git > Установить > Включить
3. В самой левой вертикальной панели появится новая кнопка "Open Git source control" > Нажимаем.
4. Открывается правая панель с управлением Git. Единственное, что там понадобится это кнопка Pull (6-я по порядку слева).
5. *Неточно*: Открытие управления может быть и ненужным. Скорее всего плагин будет сам скачивать изменения при открытии хранилища, но всё равно полезно знать, если понадобится подтянуть файлы без закрытия хранилища
## Путь второй - git pull - Точный и надёжный
1. Тут снова будет нужен cmd.
2. Открываем в консоли уже саму папку хранилища.
3. Вводим `git pull`, чтобы получить изменения.
4. Если есть желание можно создать bat файл, который при запуске будет сам открывать папку и подтягивать изменения, но с этим уже сами.
## Различия
1. Первый путь подходит, если не хочется заморачиваться с этим.
2. Второй путь *лично для меня* выглядит более привлекательным, потому что ты самостоятельно подтягиваешь изменения, и происходит всё контролируемо.
